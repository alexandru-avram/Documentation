## Numeric Data Types
### Common:
- INT: Integer, typically 4 bytes, stores whole numbers.
- FLOAT: Approximate number with floating decimal points.
- DECIMAL: Exact number with fixed decimal points, useful for financial data.

#### MySQL:
- TINYINT: 1-byte integer, range: -128 to 127.
- BIGINT: 8-byte integer, range: -2^63 to 2^63-1.
- DOUBLE: Double-precision floating-point.

#### MS SQL:
- SMALLINT: 2-byte integer, range: -32,768 to 32,767.
- NUMERIC: Same as DECIMAL, used for exact numeric values.

#### Oracle SQL:
- NUMBER: Flexible precision, stores both integers and floating-point numbers.
- BINARY_FLOAT: Single-precision floating-point.

#### PostgreSQL:
- SERIAL: Auto-incrementing integer.
- NUMERIC: Arbitrary precision number, can store large exact numbers.

#### SQLite:
- INTEGER: Stores integer values, size varies (1-8 bytes depending on value).
- REAL: 8-byte floating-point number.

#### MariaDB:
- MEDIUMINT: 3-byte integer, range: -8,388,608 to 8,388,607.
- DOUBLE: Same as in MySQL, double-precision floating-point.

#### Google BigQuery:
- INT64: 8-byte integer, range: -2^63 to 2^63-1.
- FLOAT64: Double-precision floating-point.
