## Numeric Data Types
- INT: Integer, typically 4 bytes, stores whole numbers.
- FLOAT: Approximate number with floating decimal points.
- DECIMAL: Exact number with fixed decimal points, useful for financial data.

#### MySQL:
- TINYINT: 1-byte integer, range: -128 to 127.
- BIGINT: 8-byte integer, range: -2^63 to 2^63-1.
- DOUBLE: Double-precision floating-point.

#### MS SQL:
- SMALLINT: 2-byte integer, range: -32,768 to 32,767.
- NUMERIC: Same as DECIMAL, used for exact numeric values.

#### Oracle SQL:
- NUMBER: Flexible precision, stores both integers and floating-point numbers.
- BINARY_FLOAT: Single-precision floating-point.

#### PostgreSQL:
- SERIAL: Auto-incrementing integer.
- NUMERIC: Arbitrary precision number, can store large exact numbers.

#### SQLite:
- INTEGER: Stores integer values, size varies (1-8 bytes depending on value).
- REAL: 8-byte floating-point number.

#### MariaDB:
- MEDIUMINT: 3-byte integer, range: -8,388,608 to 8,388,607.
- DOUBLE: Same as in MySQL, double-precision floating-point.

#### Google BigQuery:
- INT64: 8-byte integer, range: -2^63 to 2^63-1.
- FLOAT64: Double-precision floating-point.

## String Data Types
- VARCHAR(): Variable-length character string.
- CHAR: Fixed-length character string.
- TEXT: Large amount of text data, varying lengths.

#### MySQL
- TINYTEXT: Up to 255 characters.
- LONGTEXT: Up to 4GB of text data.
- BLOB: Binary Large Object for binary data.

#### MS SQL:
- NVARCHAR: Variable-length string, supports Unicode.
- NTEXT: Large text data, supports Unicode (deprecated).
- VARCHAR(): Stores large text data up to 2GB.

#### Oracle SQL:
- CLOB: Character Large Object for large text data.
- NVARCHAR2: Variable-length Unicode string.

#### PostgreSQL:
- TEXT: Unlimited length text string.
- BYTEA: Binary data (like BLOB).

#### SQLite:
- TEXT: Text data, length dynamically managed.
- BLOB: Binary Large Object, stores binary data.

#### MariaDB:
- VARCHAR: Same as MySQL.
- LONGTEXT: Same as MySQL.

#### Google BigQuery:
- STRING: Unicode character string.
- BYTES: Stores binary data.

## Date/Time Data Types
- DATE: Stores date values (year, month, day).
- TIME: Stores time values (hour, minute, second).
- DATETIME: Stores both date and time values.

#### MySQL:
- TIMESTAMP: Stores date and time, including timezone.
- YEAR: Stores a year in 2 or 4 digits.

#### MS SQL:
- DATETIME2: Extended DATETIME with higher precision.
- SMALLDATETIME: Stores date and time with lower precision.

#### Oracle SQL:
- TIMESTAMP WITH TIME ZONE: Stores date and time with timezone.
- INTERVAL: Stores a period of time.

#### PostgreSQL:
- TIMESTAMP: Stores date and time without timezone.
- INTERVAL: Stores a period of time, such as days or hours.

#### SQLite:
- DATETIME: Stores date and time as text, real, or integer.
- JULIANDAY: Stores date as Julian Day number.

#### MariaDB:
- DATETIME: Same as MySQL.
- TIMESTAMP: Same as MySQL.

#### Google BigQuery:
- DATETIME: Stores date and time without timezone.
- TIMESTAMP: Stores date and time with timezone.

## Boolean Data Types
- BOOLEAN: Stores TRUE or FALSE values.

#### MySQL:
- TINYINT(1): Commonly used as a boolean, where 0 is false and 1 is true.

#### MS SQL:
- BIT: Stores 0 or 1, equivalent to boolean.

#### Oracle SQL:
- NUMBER(1): Often used to store boolean values, where 0 is false and 1 is true.

#### PostgreSQL:
- BOOLEAN: Direct support for boolean values.

#### SQLite:
- INTEGER: Used to store boolean values, where 0 is false and 1 is true.

#### MariaDB:
- BOOLEAN: Alias for TINYINT(1). Commonly used as a boolean, where 0 is false and 1 is true.

#### Google BigQuery:
- BOOL: Direct boolean type with TRUE or FALSE values.

## Binary Data Types
- BLOB: Stores binary data, size may vary.
- VARBINARY: Variable-length binary data.

#### MySQL:
- TINYBLOB: Small binary object up to 255 bytes.
- LONGBLOB: Large binary object up to 4GB.

#### MS SQL:
- BINARY: Fixed-length binary data.
- VARBINARY(MAX): Variable-length binary data up to 2GB.

#### Oracle SQL:
- RAW: Variable-length binary data.
- BLOB: Stores binary large objects.

#### PostgreSQL:
- BYTEA: Binary data type for variable-length binary strings.

#### SQLite:
- BLOB: Stores any type of binary data, up to 2GB.

#### MariaDB:
- VARBINARY: Same as MySQL. Variable-length binary data.
- BLOB: Same as MySQL. Stores binary data, size may vary.

#### Google BigQuery:
- BYTES: Stores binary data of arbitrary length.

## Special Data Types
- JSON: Stores JSON formatted data.

#### MySQL:
- ENUM: String object with a set of predefined values.
- SET: Similar to ENUM but allows multiple values.

#### MS SQL:
- XML: Stores XML formatted data.
- GEOGRAPHY: Stores spatial data for geographic features.

#### Oracle SQL:
- XMLTYPE: Stores XML data.
- SDO_GEOMETRY: Stores spatial data.

#### PostgreSQL:
- JSONB: Stores JSON data in a binary format for faster querying.
- ARRAY: Stores arrays of any data type.

#### SQLite:
- NONE: Used when the column type is dynamic, and any type of data can be stored.
- JSON: Text-based storage of JSON data (available in newer versions).

#### MariaDB:
- JSON: Stores JSON formatted data.

#### Google BigQuery:
- GEOGRAPHY: Stores geographical data.
- STRUCT: Stores nested records.
